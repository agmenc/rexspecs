Tables Define Tests

Text like this is nice but decorative: it doesn't get executed. That means it isn't part of the living document*. Less is more.

* A living document is one that stays up-to-date, instead of becoming irrelevant almost as soon as the first line of code is written (which is the fate of most documents)

Things to note:
  • The first cell, in the table header, defines the type of test
  • The column headers are parsed for correctness (and the order matters)
  • Each row consists of inputs and expected outputs
  • Each row results in an API call
  • The API call returns a result, which is written back into the output document
