Tables Define Tests

Text like this is nice but decorative: it doesn't get executed. Living documents are forced to stay up-to-date, because your tests fail if they are wrong.

Things to note:
* The first cell, in the table header, defines the type of test
* The column headers are parsed for correctness (and the order matters)
* Each row consists of inputs and expected outputs
* Each row results in an API call
* The API call returns a result, which is written back into the output document
